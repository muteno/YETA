# 설정 UI 기능 계약 — 디자인 세션 인수 문서 (260707)

> 대상 = 설정/프로필 목업 만드는 디자인 세션. **디자인(레이아웃·결·모션)은 전부 네 재량** — 아래는 바꾸면 기능이 죽는 **배선 계약**과, 이미 라이브인 **권한 규칙**만 정리한 것. 시즌 정책 패널의 현 UI(`#ysetting`)는 임시 기능 정본이라 네 목업 결로 갈아엎어도 된다(계약만 지키면).

## 1) 계층별 설계 지침 — 이 표가 설계의 전부다 (라이브 — main 머지 완료)

> 정책은 3계층이고, **계층마다 "누가 만지는가"가 다르다.** 화면을 계층 단위로 나눠 그리면 로직을 몰라도 안전하다.

| 계층 | 내용 | 누가 조정? | UI 지침 |
|---|---|---|---|
| **L0 코어** — 품질·존립 | 4분류: ⓐ절대선(미성년·자해조장·실존비방·범죄조력) ⓑ출력 계약(기억 블록·MOOD) ⓒ메타 차단 ⓓ헛소리 방지 | ⓐⓑ = **아무도 못 바꿈**(레포 커밋만) · ⓒⓓ = **슈퍼관리자(2486)만** 사용/해제 토글 | ⓐⓑ = 🔒 잠금 목록으로 항목만 나열(토글 UI 절대 금지) · ⓒⓓ = 관리자에게만 2단 토글, 게스트는 잠금 표시 |
| **L1 시즌** — 수위·금기 | 4축 × 3단: 성적 표현(순한맛/기본/성인) · 욕설(순화/기본/거침) · 폭력(완곡/기본/직접) · 어두운 주제(회피/기본/깊게) | **슈퍼관리자(2486)만** | 관리자 = 3단 세그먼트 · 게스트 = 같은 화면을 read-only(현재값은 보임 · 비활성) |
| **L2 캐릭터** — 성향 | 캐릭터별 게이지 16축(말수·플러팅 등) | **누구나**(게스트 포함) | 설정 화면이 아니라 **캐릭터 카드의 ⚙**가 담당 — 설정 화면엔 안내만 |

- **2486 = 슈퍼관리자 키** · 실제 판정은 Cloudflare env `YETA_PIN_ADMIN`(레포 무노출 — 화면·코드에 PIN 숫자를 절대 쓰지 마라)
- **7890 = 게스트** · 판정 = `apps/yeta/users.json` 해시(사용자 추가 = 이 파일에 한 줄 커밋)
- 잠금 화면에서 어느 PIN을 넣었느냐로 역할이 갈리고(`YROLE`), 설정 화면은 그 역할에 따라 위 표대로 활성/잠금만 갈아 끼우면 된다.

- 판정은 전부 **서버**(op `auth`) — 뷰어에 PIN·해시 하드코딩 금지(이전 클라 해시 대조는 폐지됨).
- 언락 성공 시 전역 `YROLE`('admin'|'guest')과 `_ypin`(평문 PIN — **메모리만**, localStorage 저장 금지)이 채워진다. 새로고침 = 재입력(기존 잠금 규약).
- **UI 규칙: L0·L1 편집 컨트롤은 `YROLE === 'admin'`일 때만 활성.** 게스트에겐 잠금 표시(현행 = 🔒 + disabled 칩 + 역할 배지). 이건 UI 게이팅일 뿐이고 서버가 SET마다 PIN을 재검증하니 뚫려도 무해 — 그래도 UI는 정직하게.

## 2) API 계약 (게이트웨이 op — 전부 POST `api/yeta`)

```js
// 로그인 — 잠금 화면에서 PIN 4자리 입력 시
yApi('auth', { pin: '2486' })
// → { ok:true, role:'admin'|'guest', name:'운영자' } | { ok:false }
// 네트워크 실패·{setup:true} = 오답 아님(실패 카운트 올리지 말 것)

// 정책 정의 + 현재값 읽기 — 무인증(게스트도 열람)
yApi('policy')
// → { ok:true, def:<policy.json 전체>, p:{ sexual:2, metaban:0, ... } }

// 정책 저장 — 관리자 전용(pin 필수 동봉)
yApi('policy', { p: { sexual:2, metaban:0 }, pin: _ypin })
// → { ok:true, p } | 403 '권한 없음' | 501 'YETA_PIN_ADMIN 미설정'
// p = {key: 0~2 정수}만 — 텍스트 절대 금지(서버가 거부·라벨/문구는 policy.json이 정본)
```

## 3) 정책 데이터 구조 — **화면에 표시할 라벨·문구는 전부 `def`에서 꺼내라** (하드코딩 0 원칙)

`apps/yeta/policy.json`(= `def`)의 렌더 계약:

- `def.L1.axes[]` = 시즌 수위 4축. 각 축 `{ key, label, default, opts:[3단 라벨], hint }` → 3단 세그먼트로 렌더. 현재값 = `p[key] ?? default`. `default`와 다르면 "변경됨" 표시(현행 = 라임 도트).
- `def.L0.groups[]` = 코어 4분류. `hard:true` 2그룹(절대선·출력 계약) = **어떤 역할도 편집 UI 금지, 항목 나열만**. `hard:false` 2그룹은 `group.toggle` = `{ key, label, default:1, opts:['해제','사용'], hint }` → 관리자만 2단 토글.
- `def.L2` = 안내만(캐릭터 성향은 카드 ⚙가 담당 — 설정 화면에서 편집하지 않음).
- 저장 UX: 현행 = 800ms 디바운스 자동 저장 + "저장됨 ✓" 배지 + 실패 시 토스트. "기본값 복원" = 오버라이드 전량 제거(`p={}`) — **2탭 armed 재확인 패턴 필수**(기존 정본 `.yset-reset.armed`).

## 4) 현 임시 구현 위치 (갈아엎을 때 참조)

- 패널: `viewer/index.html` `#ysetting`(탭 패널) + `ySettingRender()` + `ySettingSave()` + `#ysettingBody` 클릭 위임 리스너.
- 목업 진입: `yShowSettings()` 맨 위 "시즌 정책" 행(`#ypolEnter`) — 현재 목업에서 유일한 실동작 행. 네 목업 안에 시즌 정책 섹션을 **직접 통합**하면 이 행·내 패널은 제거해도 됨(기능만 위 계약대로).
- 재사용 가능한 기존 정본 셀렉터: `.yset-chip(.on)`(세그먼트) · `details.yset-grp`(섹션) · `.yset-row`(행) · `.yset-saved`(저장 배지) · `.ypol-*`(역할 배지·L0 그룹 — 내가 CII 등재해둠, 갈아엎으면 CII 행도 갱신).

## 5) 하지 말 것 (기능 죽는 지점)

1. PIN·해시·정책 라벨을 뷰어에 하드코딩 (전부 서버/def에서)
2. `p`에 문자열 값 전송 (서버 거부 — enum 정수만)
3. `_ypin`을 localStorage/쿠키에 저장 (메모리만)
4. hard 그룹(절대선·출력 계약)에 토글 UI 부여 (서버에 키 자체가 없어 무동작 + 오해 유발)
5. 게스트에게 편집 UI 활성 (서버는 막지만 UX 거짓말)
