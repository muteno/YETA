# 제1명령 (포터블·완전판) — 디자인 = 루트 토큰 「계승」 또는 「갱신」

> **이 파일 하나로 어느 레포에나 이 거버넌스를 설치한다.** 사람이 아니라 **AI 에이전트가 읽고 그대로 이식**하는 스펙(누락 0·코드 포함). 15인 교차검증 반영판.
>
> **이식 순서**: (1) §1 개념 이해 → (2) §2 바인딩 표 채움 → (3) §3 파일 5개 설치(아래 코드 그대로) → (4) §4 음성테스트로 검증 → (5) §5·§6 읽고 "개선"하다 망치지 말 것.

## 0. 한 줄 요지
모든 디자인 값은 **루트 토큰을 계승**하거나(디폴트·안 물음), **운영자 승인 후 갱신**한다 — 둘 뿐. 그리고 **토큰 승인 락 게이트**가 "승인 없이 토큰 추가"를 **기계로 차단**한다. ← **이 게이트가 핵심.** 문구만 세게 박으면 과거처럼 안 지켜진다. 지켜지게 만드는 건 §3-3 게이트다.

## 1. (A) 개념규칙 — `CLAUDE.md` **최상단**에 붙일 블록 (플레이스홀더 `<…>`만 치환)

```markdown
## 🔒 절대명령 #1 (레포 제1 · 최상단) — 디자인 = 루트 토큰 「계승」 또는 「갱신」

> 위상 선언(얇게). 디자인 *내용* 축에서 이 원칙이 최우선. 실체 정본·값·절차 = <디자인계약>(충돌 시 그쪽이 이김). 이 절은 라우팅·기틀 교차검증·기존 절차를 우회하지 않는다.

값 SSOT = `<SSOT>` ("루트 디자인 토큰" = 값 SSOT 별칭). 모든 디자인 *값*(색·크기·간격·반지름·blur·모션·눌림·그림자)은 둘 중 하나:
- ① 계승 (디폴트·안 물음): 정확 토큰을 `<계승문법>`으로 → 정확한 게 없으면 가장 가까운 토큰 자동 계승.
- ② 갱신 (근접조차 없을 때만): 운영자에게 ㉮갱신할지 ㉯무엇으로 계승할지(근사 후보 제시) 물어 승인 후 진행. 값 미세조정=승인+`<GATE>` / 구조 토큰 신설=기틀=교차검증.
- 컴포넌트는 값 아님 → `<컴포넌트정본>` 정본 셀렉터 계승(별개 축).

기계강제 = `<LOCK>`(승인 원장). `<SSOT>` 새 토큰이 락에 없으면 `<GATE>` rc=1(커밋 차단). 승인 = 락 등재(PR diff 리뷰). 문구가 아니라 이 게이트가 강제한다. ⚠️ 초록 게이트 ≠ 전면 준수(아래 예외·모델 자율분 존재).

예외(raw 허용·물음 없음): 기하 계산(−50%·원 50%)·키프레임 안무 %·광학 1px 보정·OS 강제(safe-area)·로컬 z-index·토큰 없는 축(box-shadow 등).
```

> 규칙: **얇게 유지**. 값·거울·상세 절차를 여기 복붙하지 말고 `<디자인계약>`을 포인터로. (본문 3중복제 = 드리프트, 15인 지적.)

## 2. (B) 바인딩 표 — 이 슬롯만 그 레포 값으로 치환

| 슬롯 | 뜻 | yeta(참조 구현) | 새 레포에서 |
|---|---|---|---|
| `<SSOT>` | 값 단일 출처 | `viewer/index.html :root` | `:root` / `tokens.json` / Style-Dictionary / Tailwind config … |
| `<계승문법>` | 토큰 참조법 | `var(--token)` | `var()` / `theme(...)` / `$token` / `import { token }` … |
| `<LOCK>` | 토큰 승인 원장(파일) | `design-tokens.lock` | 같은 파일명 권장(레포 루트) |
| `<GATE>` | 커밋 게이트(pre-commit/CI) | `python3 shared/check_refs.py` | 그 레포의 커밋 검증 스크립트 |
| `<BUILD>` | 거울·파생 재생성(선택) | `shared/build_design_mirror.py build` | 있으면 기입, 없으면 생략 |
| `<컴포넌트정본>` | 컴포넌트 계승처 | `docs/CII_컴포넌트계승인덱스.md` | Storybook / 컴포넌트 인덱스 … |
| `<디자인계약>` | 실체 정본 절 | `CLAUDE.md §🎨` | 그 레포 디자인 규칙 절 |
| `<훅>` | 세션 주입 훅 | `.claude/hooks/design_digest.py` | 동형 SessionStart 훅 |
| `<승인자>` | operator | 레포 운영자 | 동일 |

### 2-갓차. 이식 갓차 3점 (바인딩 채우기 전 확인 · 융합 복붙 시 dangling ref)
- **방향(SSOT) 먼저 확정** — 값 소스가 CSS `:root`(yeta)인지 JSON/TS(`tokens.json`·Style-Dictionary·Tailwind theme·SCSS `$var`)인지 정한다. **후자면 CSS 는 *생성물*이라 거울 방향이 반대**(`<SSOT>`=JSON 쪽, `<BUILD>`가 CSS 를 산출). 이걸 안 정하면 거울 모델 자체가 뒤집힌다(§3-1·§3-3 의 SSOT 파싱도 그 형식으로).
- **스택별 폴백** — 거울·게이트 스크립트가 없는 레포는 `<BUILD>`/`<GATE>`/`<LOCK>` 을 "N/A(사람 규율)"로 두되 **개념 ①②③(계승/근접 자동계승/갱신)은 여전히 유효**. "스크립트 없으면 규칙도 없다"가 아님(§3-3 게이트는 강력 권장이되, 부재 시 소프트룰로라도 강제).
- **한글 고정명 중립화** — `구성도`·`docs/CII_…` 등 한글 경로는 그 레포 언어/관례로 치환(`<컴포넌트정본>`·`<디자인계약>` 슬롯). yeta 스크립트가 한글 경로를 자동 생성할 수 있으니 타 레포 오염 주의.

## 3. 설치 — 파일 5개 (코드 그대로 이식)

### 3-1. `<SSOT>` — 토큰을 한 곳에
raw 값 산재 금지. 모든 색·크기·반지름·blur·모션·눌림·그림자 값을 토큰으로 정의.
- CSS면 `:root { --token: value; … }`.
- JSON/TS면 `tokens.json` / `tokens.ts`(Style-Dictionary·Tailwind theme 등).

### 3-2. `<LOCK>` = `design-tokens.lock` — 승인 원장 (레포 루트)
현재 토큰명을 **전부** 부트스트랩 등재(= 기존분은 승인된 것으로 간주). 형식:
```
# design-tokens.lock — 운영자 승인 토큰 원장 (절대명령#1)
# <SSOT> 의 모든 토큰명은 여기 등재돼야 함. 미등재 신규 = <GATE> rc=1 (커밋 차단).
# 승인 = 여기 등재(PR diff 를 운영자가 리뷰). 형식: 1줄 1토큰(주석=사유).
--fg
--accent
--r-m
…(현재 토큰 전부)
```

### 3-3. `<GATE>` — token-lock 체크 ★핵심★ (아래 코드 그대로)
```python
# <GATE> 스크립트에 추가 · 미승인 신규 토큰이면 rc=1 반환 → 커밋 차단
import os, re
def check_token_lock(SSOT_PATH, LOCK_PATH):
    if not os.path.exists(LOCK_PATH):
        print('⚠️ token-lock skip — lock 없음'); return 0
    src = open(SSOT_PATH, encoding='utf-8').read()
    # ── SSOT 에서 토큰 '정의명'만 추출 ──
    #   CSS :root  → 아래 그대로.  JSON/TS  → 이 3줄을 키 파싱으로 교체(예: json.load(...).keys()).
    m = re.search(r':root\s*\{(.*?)\n\s*\}', src, re.S)
    root = re.sub(r'/\*.*?\*/', '', (m.group(1) if m else src), flags=re.S)  # 주석 제거(오탐 가드)
    cur  = set(re.findall(r'(--[A-Za-z0-9-]+)\s*:', root))
    # ── 락 원장 토큰명 ──
    lock = set(re.findall(r'^\s*(--[A-Za-z0-9-]+)\b',
                          open(LOCK_PATH, encoding='utf-8').read(), re.M))
    new, gone = sorted(cur - lock), sorted(lock - cur)
    if new:
        print('❌ token-lock (BLOCK·절대명령#1) — 미승인 신규 토큰:')
        for n in new: print('  -', n, '→ 운영자 승인 후 lock 등재(임의 추가 금지)')
        return 1
    if gone: print('⚠️ token-lock — 락에 있으나 SSOT 부재(삭제됨→정리):', *gone)
    print('✅ token-lock — 토큰 %d개 전원 승인(미승인 신규 0)' % len(cur))
    return 0
```
> 비-CSS SSOT(JSON/TS): `cur` 추출만 그 형식에 맞게 바꾸면 됨(락 형식·집합 차 로직은 동일). Node면 같은 로직을 JS로 포팅(정규식 2줄 + Set 차).
> **게이트 오케스트레이터에 반드시 배선**: `if check_token_lock(...) != 0: rc = 1`. pre-commit 훅이 `<GATE>` 를 호출하게(설치돼 있지 않으면 `.githooks/pre-commit` + `git config core.hooksPath .githooks`).

### 3-4. `<훅>` — 세션 주입 (선택이지만 강력 권장)
SessionStart(및 UI 작업 턴)에 §1 요지를 **자동 주입** → 압축 세션·긴 대화에도 계약 생존(사람이 매번 안 붙여도 됨). 주입 문구:
```
[🔒 절대명령#1 — 디자인 = 루트 토큰 「계승」 또는 「갱신」 (레포 제1·최상단 · <디자인계약> §🔒)]
· 계승(디폴트·안 물음): 정확 토큰 <계승문법> → 없으면 가장 가까운 토큰 자동 계승.
· 갱신(근접조차 없을 때만): 운영자에게 "갱신할지 / 뭘 계승할지(근사 후보)" 물어 승인 후. 구조 토큰 신설=기틀=교차검증.
· 기계강제: <LOCK> — <SSOT> 새 토큰이 락 미등재면 <GATE> 커밋 차단(승인=락 등재·PR diff).
· 예외(raw 허용·물음X): 기하·키프레임 안무·광학1px·OS(safe-area)·로컬z·토큰없는 축(box-shadow). 초록 게이트≠전면 준수.
```
> ⚠️ 훅과 CLAUDE.md를 **같이** 갱신(한쪽만 고치면 두 소스 드리프트 = 15인 지적).

## 4. 검증 (필수) — 음성테스트로 "진짜 막히는지" 확인
설치 후 반드시:
1. `<SSOT>` 에 가짜 토큰 1개 임시 추가 (예: `--zzztest: 1px;`).
2. `<GATE>` 실행 → **`❌ token-lock (BLOCK)` + 종료코드 1 이어야 정상**(안 막히면 배선 실패).
3. 가짜 토큰 제거 → `<GATE>` 재실행 → 종료코드 0.
> 이 테스트가 통과해야 "지켜지게" 가 실제로 작동하는 것. (yeta에서 실측: 주입 시 rc=1, 복원 시 rc=0.)

## 5. 왜 이 구조인가 (15인 교차검증 수렴 · AI는 이 의도를 지켜라)
- **우선권 무한정 금지** — "다른 모든 규칙에 우선"은 기틀 교차검증·라우팅을 우회하는 라이선스가 된다 → "디자인 *내용* 축 최우선, 충돌 시 실체 정본이 이김"으로 스코프.
- **기계강제가 유일한 진짜 지렛대** — 선언만 세게 하면 안 지켜진다(과거 반복). `<LOCK>` 게이트 = "승인의 *부재*를 기계가 잡음".
- **근접 자동계승 보존** — ①정확→②근접 자동(안 물음)→③갱신(물음). 이 중간티어가 빠지면 물음이 폭증 = 마비.
- **예외 카브아웃 필수** — "전부 절대금지·둘 뿐"은 실측상 거짓(기하·box-shadow 등 정당 raw 대량). 예외 없으면 마비.
- **얇게 + 훅 동기** — §디자인계약·훅과 3중복제하면 드리프트. 최상단=얇은 선언+포인터, 실체는 한 곳.

## 6. 하지 말 것 (안티패턴 — AI가 흔히 망치는 지점)
- ✗ "모든 규칙에 우선/무효" 같은 무한정 절대어(우선권 전쟁·기틀 우회 유발).
- ✗ `<LOCK>` 게이트 없이 §1 문구만 삽입(= 강제 0 = 과거처럼 안 지켜짐).
- ✗ 기하·키프레임·box-shadow까지 토큰화 강요(드리프트 폭발·마비).
- ✗ ②갱신 시 락 등재만 하고 운영자에게 안 물음(락 diff가 PR 리뷰 대상이란 점이 승인 지점).
- ✗ CLAUDE.md만 고치고 `<훅>` 방치(두 소스 드리프트).
- ✗ "게이트 초록 = 완벽 준수"로 착각(예외·모델 자율분은 게이트 밖).
