## 3. 팩 개선 제안 (이식하며 부딪힌 것 — 핵심 피드백)

### ⚠️ A. 슬롯 S1/S2 "없음" 기본답이 디자인-보유 레포에서 함정
- **증상**: 슬롯 답으로 S1(디자인 정본)=없음, S2=CLAUDE.md만 받았는데, YETA는 강한 디자인 정본(:root 56토큰·절대명령2 인덱스·CII·게이트)이 **이미 있는** 레포였다. 슬롯대로 갔으면 `[4]`/`[9]`에서 디자인 포인터가 `[미정]`으로 비어 레포 실측과 정면 모순.
- **제안**: 슬롯 안내에 한 줄 — *"이식 대상 레포에 디자인 정본이 실재하면 S1/S2는 '없음' 금지, 실제 경로 필수."* 또는 이식 세션이 레포의 디자인 정본 존재(예: `:root` 토큰·컴포넌트 인덱스·게이트)를 **자동 감지해 슬롯 답과 대조**하고 불일치 시 운영자에게 되묻는 절차.

### ⚠️ B. §🔒/§🎨 "반입 금지"가 소스 레포에선 dangling을 만든다
- **증상**: 팩은 §🔒/§🎨 디자인 콘텐츠를 "반입 금지(타 레포)"로 스킵. 근데 YETA(정본 소스 레포)는 **다른 문서**(절대명령2 인덱스·`design_digest` 훅·CII·공통규약)가 이미 `CLAUDE.md §🔒/§🎨`를 참조 중이었다 → 팩 골격으로 덮으니 그 섹션이 사라져 **참조가 전부 dangling**이 됐다(이식 전부터 잠재, 골격 교체가 표면화).
- **제안**: 반입 금지 규정에 단서 — *"반입 금지 = 타 레포로의 반출 제약. **정본 소스 레포**(다른 문서가 CLAUDE.md §섹션을 참조하는 레포)에 이식할 땐 그 §섹션을 **얇은 포인터로 보존**하라(삭제 ≠ 반입금지)."* 이식 체크리스트에 "레포 전역에서 `CLAUDE.md §<앵커>` 참조를 grep → 골격이 접는 섹션이 있으면 얇은 포인터로 유지" 추가.

### ⚠️ C. 완료 정의 "main 머지 + git show origin/main"이 하네스 강제와 충돌
- **증상**: 실행 환경이 지정 브랜치 + 드래프트 PR을 강제하고 main 직push를 막았다. 팩 완료 기준(`main 머지 후 read-back`)을 그대로 못 쓴다.
- **제안**: 완료 정의를 *"정본 브랜치(main **또는 세션 지정 브랜치**) 머지/푸시 + `git show origin/<정본브랜치>:<파일>` read-back"*으로 일반화. 드래프트 PR을 승인 관문으로 인정.

## 4. 팩이 잘 작동한 것 (유지 권장)
- **`[1]~[14]` 원문 보존 = diff로 검증** — 충실도 기준(diff=슬롯뿐)이 명확해 자가검증이 3초. 강력.
- **"사라질 규칙 목록" 사전 승인 게이트** — 이게 없었으면 위 A(디자인 포인터 소실)를 놓쳤다. 회귀를 실제로 잡은 안전장치. 유지.
- **`[6]` 정본 예시 placeholder 흐름** — "첫 완료 보고 후 후속 커밋으로 채움"이 시간 역전 없이 깔끔.
- **첫 이식 커밋 `[9]` 평의회 면제** — 팩 완료기준(diff·반입금지·read-back)이 대체. 합리적.
- **디자인방식론 팩 별도 이식** — 값 축/방식론 축 분리가 명확해 중복(YETA 기존 정본과) 회피가 쉬웠다.
