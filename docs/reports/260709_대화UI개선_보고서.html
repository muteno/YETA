2. 왜 이렇게 (근본 원인)

배경 잔상·인트로 영상
시즌 감정 미디어 base[0]이 mp4 클립이라 답장수 0(첫 진입·재진입)마다 클립이 첫 페인트가 됐고, 무대 2겹 레이어는 챗을 닫아도 소거되지 않아 멈춘 프레임이 다음 진입에 비쳤다. → 진입 = 이미지 우선 + 방 전환/닫힘 = 레이어 전체 소거로 근본 차단.
토큰 미터
러너는 이미 claude_meter로 usage를 계측하고 있었지만 세션 턴에는 안 실렸다. → 미터 래퍼에 옵트인 METER_LAST 출구 1개 추가 → 러너 finish()가 답장 턴에 tok{i,o} 박제 → 뷰어 합산. 파이프라인 호출·출력 계약 무변경(fail-soft — 계측 실패 = tok 미박제).
설정 안 먹힘
read/call/ptt 토글은 저장만 되고 소비처가 없었고, 모델·글자 크기는 read-only 행이었다. → 소비처 배선(yRender·call.js onSess·html 클래스) + 다이얼/배율 행 신설. 게이트웨이 MODELS/EFFORTS 화이트리스트는 원래부터 sonnet-5·max를 허용 = 서버 무수정.
전화 락 오독
수화기와 자물쇠가 같은 색(mut)에 자물쇠만 40% 투명 — 겹쳐 보여 "열쇠 꽂힌" 실루엣으로 읽혔다. → 자물쇠만 터콰이즈(--think 기존 토큰) 불투명으로 분리. 신규 색 0.

4. 리스크·트레이드오프

  • 주의과거 대화 토큰은 소급 불가 — 미터는 이번 배포 후 답장부터 누적(러너가 그때부터 박제).
  • 주의글자 크기 = 토큰 런타임 배율 — :root 원값 스냅샷(6종) 기반이라, :root 타이포 토큰 값을 바꾸면 YFS_BASE도 같이 갱신 필요(주석 명기).
  • 폰트 1회 마이그레이션은 260708 이전 저장분('sys')만 소거 — 이후 사용자가 다시 끄면 그대로 존중.
  • 시즌 클립은 폐기 아님 — 대화 진행 중 로테이션에서 재생(진입 프레임만 이미지 우선).
  • '일일 통화 상한'·'테마'는 서버 env·브랜드 기틀이라 read-only 유지(사유 표기). '답장 알림'은 권한 요청까지(웹푸시 구독 저장 축은 별도 백엔드 필요 = 후속).
