유저 프로필 — “AI가 나를 부르는 법”

호칭 + 소개를 설정하면 캐릭터가 실제 대화에서 그 이름으로 부른다 · 118차 · 260708 · PR #192
한 줄 요약. 프로필 화면에서 내 호칭(이름)과 소개를 적으면, 세션에 저장돼 매 답장·오프닝·초대·넛지 프롬프트에 비신뢰 격리 블록으로 주입되고, 캐릭터가 그 이름으로 나를 부른다. 무인증 공개 게이트웨이라 클라 텍스트는 고정점 정화 + “지시 아님” 격리로 프롬프트 주입을 원천 차단.

전 / 후

전
내 프로필의 이름·닉네임·상태 메시지는 localStorage 전용 — 게이트웨이 미전송, 대화에 전혀 반영 안 됨. 캐릭터는 나를 늘 “너/유저”로만 지칭.
후
프로필에 “AI가 나를 부르는 법” 섹션(호칭 + 소개) → 저장 시 op me로 서버 동기 → 러너가 매 턴 주입 → 캐릭터가 “지호야, …”처럼 호명. 기존 대화도 다음 턴부터 소급 반영.

파이프라인

뷰어 프로필 → op me(stripMarkers+캡·CAS) → R2 세션 me:{call,about}(전 방 공유) → 러너 me_block(재정화+격리 프레임) → 프롬프트 [유저 프로필] 블록 → 캐릭터 답장
