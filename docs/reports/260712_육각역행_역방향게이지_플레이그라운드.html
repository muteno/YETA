육각 역행 게이지 — 발전가능성의 역방향

흰 점선(발전 가능성·상방)의 대칭으로 빨간 점선(역행·하방)을 base 안쪽에 같은 점선·투명도 겹침으로. 축마다 늘고(튀어나가고) 줄고(파이고) 다르게.

3층 육각 — 현재 2층(발전+현재)에 역행 1층을 대칭 추가:

층의미표기토큰(계승)

1 인터랙티브 — 축별로 늘리고 꺾어보기

발전 가능성 (base 바깥 · 흰 점선) 현재 유효 스탯 (라임) 역행 (base 안쪽 · 빨간 점선)

축 슬라이더: + = 발전(바깥으로) · − = 역행(안쪽으로) · 0 = base

2 전 · 후

지금 (2층 — 발전만)

좋아질 여력만 흰 점선. 나빠짐은 안 보임.

제안 (3층 — 발전 + 역행)

역행하면 빨간 점선이 안쪽으로 파인다.

3 리스크 · 판단 포인트

빨강 = 오류 오독 우려: 앱에서 --danger 빨강은 에러·재시도에 쓰던 색이라, 육각의 빨간 음영이 "고장/경고"로 읽힐 수 있다(평의회 UX 위원 지적). 운영자 명시 지시 = 빨강이라 이 시안은 --danger 계승으로 갔다. 대안: 역행을 --mut 탈채도(무채색 파임)로 하면 "냉담"으로 읽히고 오류 오독이 없다 — 프리셋 [대안: 무채색]로 비교해봐.
기하 예외: 점선 dasharray·폴리곤 좌표는 토큰 없는 기하축(계산값) — §🎨 raw 허용. 색만 --danger/--mut 토큰 계승. 신규 색 창작 0.
배선: yHexSvg(stats, big, plus) → yHexSvg(stats, big, plus, minus) 확장. minus = 역행 스탯(min(minus, base) = base 안쪽만). 신규 CSS 2줄 .hx-n(빨간 음영)·.hx-no(빨간 점선 둘레). 역행값 소스 = 역행 v2의 yHurt(활성 상처가 온기·장난·텐션 축을 base 밑으로).
