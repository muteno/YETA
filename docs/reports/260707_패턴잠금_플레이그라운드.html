재현 한계·주의
  1. 배경 = 실물 영상 히어로(lock-hero.mp4) 대신 다크+라임 톤 근사. 실제 앱은 캐릭터별 잠금 배경 위에 얹힘.
  2. 번호 유지 매핑: 3×3 점 = 전화 키패드 순서(1–9). 그린 순서 = PIN 문자열 → 서버 auth op·기존 번호 무변경. 변환은 클라이언트에서만.
  3. 0 포함/중복 숫자 PIN: 3×3엔 0이 없고 표준 패턴은 한 점 재방문 불가 → 0·반복 숫자 PIN은 운영자 결정 필요(예: 중앙=0 매핑, 또는 재방문 허용, 또는 해당 PIN만 키패드 폴백).
  4. 데모 검증(2486)은 로컬 대조 시연 — 실제는 서버 판정(admin=env·guest=users.json).
  5. ⚠ 잠금 화면 = PIN 게이트(기틀 인접). 실제 반영은 §기틀 보호 확인 + 토큰 없는 패턴 기하는 raw 예외축, 색·글래스·radius는 var() 토큰 계승.
