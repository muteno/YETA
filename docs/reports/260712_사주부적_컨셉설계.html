사주 부적 — 사주를 받아 맞춤 부적을 지어주는 기능

유저의 사주(생년월일시)를 받아 해석하고, 그 결과로 그 사람만의 부적(이미지+문구)을 지어준다. 무음동의 오래된 존재가 손수 써주는 서사.

위상 = 캐릭터챗 위 얹는 신규 축(기틀 = 운영자 승인·5인 검증 후 구현) · 이미지 파이프라인 = 기존 yeta_face 패턴 재사용 · 이 문서 = 컨셉 설계

왜 무음동에 맞나. 무음동 = 경계가 얇은 골목·각성자·도깨비·몇백 년 산 찻집 주인. 사주·부적은 이 판타지 톤과 정확히 맞물린다 — 점집이 아니라 "이 동네의 신비한 존재가 네 기운을 읽고 지켜줄 부적을 써주는" 서사. 시즌 특전(--season 보라 축)으로 얹기 좋다.

핵심 = 개인화. 얼굴 프로필(meface)처럼, 사주 부적도 유저 고유 입력 → 고유 산출물. 재미·소장 가치·공유 훅.

1 유저 플로우 (4단)

STEP 1
사주 입력
생년월일 + 태어난 시(모르면 '시 모름') + 양/음력. 뷰어 폼 · 한 번 입력 = me.saju 저장(재입력 불요)
STEP 2
사주 해석
간지 변환 → 오행 균형·십신·용신 산출(규칙 계산) → 캐릭터가 "네 기운"을 읽어 풀이(LLM·판타지 톤)
STEP 3
부적 생성
해석 결과(부족한 오행·용신) → 맞춤 부적 이미지(전통 부적 스타일) + 축원 문구 생성
STEP 4
부적 카드
부적 이미지 + 풀이 + 문구를 카드로 · 저장·공유 · 무음동 존재의 서명

2 아키텍처 (기존 배관 재사용)

층설계재사용
입력·저장뷰어 폼 → op saju(게이트웨이) → sess.me.saju{생년월일시·양음력} CAS 저장. 텍스트 0 클라 노출(meface 패턴)me op·프로필 패턴
사주 계산규칙 계산(결정적): 만세력 간지 변환·오행 카운트·용신 = .github/scripts/yeta_saju.py(무과금·결정적 · LLM 아님 = 정확성)yeta_place.py(결정적 판정) 패턴
해석 풀이계산 결과 + 캐릭터 카드 → claude -p(판타지 톤 풀이) = 답장 파이프 재사용yeta_chat.sh dispatch
부적 이미지용신·오행 → 프롬프트 조립 → gpt-image(전통 부적 스타일·붉은 주사·전서체 느낌) → 공개 R2 yeta_talisman/<uid>.pngyeta_face.py 패턴(방금 픽스한 주입 로직)
트리거op talisman(⚠️유료 가드 = 일 상한 YETA_TALISMAN_MAX_PER_DAY 기본 1~2 · meface 패턴) or 수동 dispatchmeface 유료 가드 축

계산은 규칙·풀이는 LLM. 사주 간지·오행은 결정적 규칙으로(만세력 = 수학) — LLM에 맡기면 틀린다(십신·용신 계산은 명확한 알고리즘). LLM은 그 결과를 캐릭터 톤으로 풀어주는 역할만. yeta_place.py가 동선을 결정적으로 판정하는 것과 같은 분리.

3 누가 지어주나 (캐릭터 연계)

후보 A — 홍석천(찻집 주인·9w1): 몇백 년 산 존재·판단 없이 품는 리스너. "네 기운을 읽고 지켜줄 걸 써줄게" = 자연스러움. 기존 캐릭터라 신규 0.

후보 B — 신규 시즌 캐릭터(역술가·무당): 부적 전담 캐릭터(시즌 특전). 세계관 확장이나 신규 캐릭터 = 기틀.

후보 C — 캐릭터 무관 앱 기능: "무음동 부적소" 기능으로 서사 최소. 가장 가벼움.

추천 = A(홍석천) — 서사 정합·신규 0·시즌 없이 즉시. 운영자 택.

5 안전·품질 가드 필수

운세 = 오락·엔터, 단정 예언 금지(L0 정합). ① 의료·수명·죽음·질병 단정 금지 ② 금전·투자·도박 구체 조언 금지 ③ "반드시·확정" 류 운명 단정 금지 → "~기운이 강한 편·~을 조심하면 좋아" 완곡·긍정 프레임 ④ 불안 조장 금지(부적은 지켜주는 것·저주·액운 강조 X) ⑤ 실존 종교·무속 비하 금지. = policy L0 존엄·품질 축에 편입.
부적 이미지 안전: 전통 부적 스타일(붉은 주사·상징)로 tasteful · 실제 종교 부적 위조·특정 종교 성물 모사 금지 · "재미·판타지 소품" 톤 유지.
