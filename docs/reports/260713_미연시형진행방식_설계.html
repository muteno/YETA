말벗 제타 — 미연시형 진행 방식 설계

기존 LV·해금·yBond·마주침 축 위에 〔루트 · 이벤트 · 엔딩 · CG해금 · 선택지 · 진행 가시화〕를 얹는다 — 엔진 신설이 아니라, 매 턴 이미 배출되는 관계기억을 읽어 채팅 밖에 그리는 것.

설계 전용 · 구현 0 컴패니언 챗 정체성 불변 비용 불변 · 2번째 dispatch 0 자유대화 대원칙 L0/policy 불가침 근거 = 레포 실측 + 평의회 3인

0.한 줄 비전 — “관계의 침전물”

“메뉴도 세이브도 게임오버도 없이 — 매 턴 공짜로 배출되는 관계기억(NOTE:ME의 [LV]·[★]·[사건]·MOOD)을 ‘결(grain)’로 읽어, 자유대화는 한 글자도 안 좁힌 채 루트·사건·칭호·CG·스탯을 관계의 침전물처럼 채팅 밖(프로필·앨범·지도)에 떠오르게 하는, ‘내가 대해온 방식이 곧 우리의 도달점’인 미연시.”

엔진을 새로 만들 필요가 없다. 이미 dispatch 1회 안에서 모델이 관계 상태를 배출 중이다 — 이건 돌아가는 진행 기계다. 설계의 정체 = ㉮ 배출물을 살짝 더 풍부하게(같은 dispatch·서브라인 몇 줄) + ㉯ 결정론으로 읽어(yBond·places.py·날짜연산·CSS — LLM 0회) + ㉰ 채팅 밖에 그린다(프로필·앨범·지도). 대화는 배경 레이어만 얹히고, ‘진행하는 맛’만 남는다.

2.설계 대원칙 — 살릴 것 / 버릴 것

적대적 실현가능성 감사(READ-ONLY·레포 실측) 결론: 미연시를 ‘엔진으로 신설’하면 비용·자유대화·보안 제약을 정면으로 깬다. ‘기존 축의 변형·강화’로 재정의하면 대부분 합법.

✓ 살린다 (컴패니언 챗과 양립)

  • LV 게이트 이벤트 — 결정적·무LLM 트리거(barge_check 패턴)
  • 정적 CG의 클라이언트 해금 — 미리 만든 에셋을 단계 도달 시 개방
  • 모델 연출형 서사 정점 — LV5+ 관계의 절정(열린 상태 유지)
  • 스레드=루트 · yBond(게이트)+LV(연출) 이중 플래그
  • 침묵·부재를 관계 신호로 — 자유대화 챗만의 고유 강점

✕ 버린다 (정체성 붕괴 요인)

  • 메뉴형 A/B/C 선택지 — 자유대화를 스크립트로 경직
  • 상호배타 강제 루트(A 고르면 B 잠금) — 방 독립·기억 공유와 충돌
  • 비가역 터미널 엔딩(game-over·방 닫힘) — 지속 관계와 충돌
  • 플래그 도달 자동 CG 생성 — 유료 종량제 = 비용 폭탄
  • 노출 수치·스탯 육성 미니게임 — 로맨스의 은닉성·톡결 파괴

레드라인 (절대 불가침 · 감사 지정)

① 진행 판정용 별도 dispatch 신설 금지 — 이벤트/루트/엔딩 판정은 barge_check처럼 결정적·무LLM이거나 그 턴의 단일 답장에 태운다(char:main 단일 그룹·스레드별 분기 금지 불변).
② 강제 메뉴·상호배타 루트 잠금 금지 — 진행은 대화 위 배경 레이어까지만, 대화를 대체하는 레일이면 안 된다.
③ 플래그 도달 자동 이미지 생성 금지 — CG는 사전 생성 정적 에셋 + 수동/상한 dispatch만, 해금은 뷰어 클라 게이트일 뿐.
④ 진행 상태를 레포 커밋 금지(R2 세션에만) · L0 출력 계약(NOTE/MOOD) 임의 개조 금지. 새 제어 마커는 stripMarkers 화이트리스트 편입 + enum/숫자화 필수(유저 위조 주입 차단).
⑤ 비가역 엔딩·선형 호감도 누적으로 관계 모델 대체 금지 — LV는 오르내리고 소원은 회복 가능. 지속 컴패니언 관계가 정본.

4.루트 — 메뉴 없이 방향을 갖기 변형

유저가 뭘 고른 게 아니라, 계속 그렇게 대해온 결(grain)이 쌓여 루트가 된다.

4-1. 결(grain) 서브라인 = 잠재 루트 기틀·핵심

NOTE:ME 첫 줄 [LV n] 아래로 [결:라이벌]·[결:다정]·[결:관찰자]·[결:정면] 한 줄을 모델이 누적 태도에서 추론해 박제(같은 dispatch·토큰 몇 개).

왜: 메뉴 0개인데 루트가 생긴다. 세라를 계속 맞받아치면 라이벌 결, 챙기면 다정 결로 해금 에피소드의 온도가 달라진다 — “내가 만든 관계”라는 미연시 최고의 쾌감.

4-2. 호칭 사다리 = 눈에 보이는 루트 마커 공짜

캐릭터가 나를 부르는 법(야/너→이름→애칭)을 [호칭:…]으로 박제, 결마다 애칭이 갈림(라이벌=“야, 맞수” / 다정=이름+부드럽게). me “나를 부르는 법”과 맞물림.

왜: 루트 진행이 호칭 한 단어로 매 턴 체감. 이미 있는 me축을 진행 게이지로 승격 — 추가 UI 0.

4-3. 초반 3~5턴 = 인상 각인창 공짜

카드 relationship이 출발점이듯 첫 몇 턴을 결 판정에 가중(STATE_BLOCK “아직 인상 형성 중” 힌트). 첫인상이 이후 결의 기울기를 크게 정함.

왜: “처음이 중요하다”는 긴장을 선택지 화면 없이. 재방문(reset)마다 다른 결 = 리플레이성.

4-4. tunes(L2)가 루트 색을 미리 칠함 공짜

유저가 tune을 따뜻하게 밀면 다정 결이, 날카롭게 밀면 라이벌 결이 도달하기 쉬운 소프트 바이어스(러너가 tune 배열 읽어 STATE_BLOCK 한 줄).

왜: 루트 주도권을 메뉴가 아니라 성향 슬라이더로 — 자유대화 대원칙과 충돌 0.

4-5. 안티루트 = 실패가 아니라 다른 결 공짜

차갑게·적대적으로 일관해도 game-over가 아니라 앙숙/라이벌/츤데레-상호 같은 정당한 관계 텍스처로 침전(yEstranged가 메커니즘).

왜: 오답이 없다. 어떤 일관된 태도든 읽히는 관계가 되니 유저가 겁 없이 자기답게 — 컴패니언 챗 정체성 그대로.

5.이벤트 — 경직 트리거 없이 ‘발견’으로 변형(핵심 지뢰)

이벤트가 “발사”가 아니라 “발견”으로 온다. 모든 트리거는 결정론·무LLM(러너) + STATE_BLOCK 주입 → 그 턴의 단일 dispatch가 연출. 2번째 dispatch 0.

5-1. STATE_BLOCK ‘오늘의 계기’ 시드 기틀-근접

날짜+bond+장소 결정론 시드가 앰비언트 훅 하나(날씨·기념일·마주침)를 기존 프롬프트에 주입(회상의 확장판). 모델은 꺼낼지 말지 스스로 판단.

왜: 경직 트리거 특유의 삐걱임 없이 사건이 자연 발생. 새 dispatch 0.

5-2. 기념일 = [사건]·[★] 타임스탬프 회귀 공짜

이미 감정가중된 [★]고백·[사건]다툼 줄에 서버가 날짜연산만 얹어(“그날로부터 딱 한 달”) 계기 힌트로 주입.

왜: 세상에 없던 나만의 기념일. 스크립트 이벤트를 유저-고유 로그로 대체 — 개인화 최상·비용 0.

5-3. 위치 마주침 × 해금 게이트 = 무대 정렬 공짜

places.py 마주침이 LV 문턱과 겹칠 때 사건으로 승격. 세라 [LV5+] “무대 위 세라”가 옥상 마주침과 정렬되면 그게 곧 CG 이벤트 무대.

왜: 있는 두 시스템(마주침+해금) 융합으로 “그 장소에서 그 일이 열렸다”는 필연. 신규 로직 최소.

5-4. 시간대 변신(mode) = 하루 리듬 이벤트 공짜

백·루시의 mode(밤/낮 변신)를 앰비언트로 노출해 밤에만 열리는 결·심야 라디오(세라가 매일 듣는) 같은 반복 이벤트를 시계로 발생.

왜: 하루 시간대가 콘텐츠가 된다. “밤에 만나러 가는” 리듬 = 미연시 시간표 감각을 공짜로.

5-5. 역행 = 위기 이벤트, 재회 = 페이오프 공짜

bond 하락이 숫자로만 끝나지 않고 “소원해진 정적” 앰비언트 주입 → 화해-마주침이 이벤트가 되고 재회가 보상.

왜: 갈등이 죽은 끝이 아니라 서사의 산. 역행을 다룰 줄 아는 미연시는 드물다 — 여기선 공짜로.

6.엔딩 — game-over 없이 ‘칭호’라는 지속 상태 재정의

6-1. 관계 칭호 = 종료 아닌 진입 상태 기틀

LV5 지속 시 NOTE:ME에 [칭호:연인]·[소울메이트]·[절친]·[스승-제자]·[필생의 맞수]·[절연]이 결정화되고 배지로 노출. 끝나는 화면이 아니라 들어가 있는 방.

왜: “엔딩 봤다”의 카타르시스를 주면서 챗은 안 죽는다. 컴패니언의 근본 제약을 미연시 문법으로 푼 지점.

6-2. 분기 = 결 × 최고 LV 교차표 공짜

도달 칭호 = (4장의 지배 결)×(최고 bond) 결정론 매핑. 라이벌 결+LV5=필생의 맞수, 다정 결+LV5=연인/소울메이트.

왜: 4장의 루트가 6장의 엔딩으로 회수되며 한 시스템으로 닫힘 — “내가 대해온 방식이 곧 도달점”.

6-3. 절연 = 정당한 엔딩 + 재회로(路) 공짜

깊은 배신은 [절연](차갑고 닫힌 텍스처)로 가되, 마주침 재회로 회복 경로가 열림. “졌다”가 아니라 하나의 결말이자 다시 여는 문.

왜: game-over 금지를 정면으로 지킴. 배신이 실패가 아니라 콘텐츠.

6-4. 회차 없이 앙상블 엔딩 공짜

방이 캐릭터별·기억 공유라 한 명과 연인·다른 명과 스승 엔딩을 동시 보유. 루트락 없고 합석이 그 공존을 보여줌.

왜: 강제 일부일처/루트락 없는 앙상블. 컴패니언의 다관계성을 엔딩까지 확장.

6-5. 에필로그 = 살아있는 회상 공짜

도달 칭호가 회상을 가끔 트리거(“우리 그때…” [★] 마일스톤 재생) = 정지된 카드가 아니라 숨 쉬는 에필로그.

왜: 회상 메커니즘 재탕으로 엔딩이 박제되지 않고 여운으로 이어짐.

7.CG 해금 — 있는 에셋을 마일스톤 보상으로 변형(무과금 우선)

7-1. 얼굴 = 실루엣 → 선명 (이미지 1장 + CSS) 뷰어

미대면 ???+실루엣이 이미 있으니, 같은 avatar에 blur/brightness 필터를 LV로 단계 해제(실루엣→흐림→선명). “처음 얼굴을 또렷이 본 순간”이 CG.

왜: 미연시 정석 “캐릭터 등장 CG”를 이미 있는 이미지 1장+CSS로. 무과금·즉시.

7-2. 무드 배경 = 감정 CG 공짜

bg가 이미 warm/tense/blue 배리언트 보유. 고백·LV업 순간에 특별 무드-bg를 “장면 CG”로 고정 노출(옥상 배경 at LV5).

왜: MOOD bg가 이미 존재 → 이벤트 CG로 용도 전환. 추가 생성 0.

7-3. 보이스 = LV 게이트 첫 수신 뷰어틀

haeun el: 음색·걸려오는 전화 파이프가 이미 있음. “처음 목소리를 듣는 순간”을 LV 마일스톤 뒤로 게이트(TTS 유료지만 일 상한이 이미 가드).

왜: “첫 보이스”는 관계 각인의 정점. 있는 음성축을 보상 타이밍으로만 재배치.

7-4. 특별 스틸 = grade 감정배경/젬 배지 공짜

lucy(season 보라 감정배경)·winter(idol 실버 젬)의 등급 비주얼을 마일스톤 해제 = “프리미엄 CG”. 이미 만들어진 에셋.

왜: 등급 아트를 스페셜 스틸 언락으로 재활용 — 완성품 재탕.

7-5. 앨범(갤러리) = CG 도감 뷰어

해제한 얼굴/무드bg/보이스/배지를 모은 갤러리, 미해제분은 실루엣 티저([LV4+] ???). 미연시의 “CG 도감”.

왜: 수집·완성 욕구를 신규 에셋 0으로. “다음 CG 뭐지”가 진행 동력(챗에 없던 회차 훅).

8.선택지 — 소프트, 메뉴 없이 원형 불가 / 변형만

  • 유저 지문이 곧 선택 공짜 — (…)·*…* = 실제 행위(운영자 260712 계약)가 이미 “선택”이다. 리터럴 A/B/C 메뉴는 폐기.
  • 비강제 제안 pill 뷰어 — UI가 선택지를 원하면 탭 시 그 텍스트를 유저 발화로 send하는 제안 pill만. 분기 잠금 없음(스킵해도 무방).
  • 결정적 순간의 무게 공짜 — 고백·배신·비밀공유 같은 관계 정의적 발화는 [★]·[사건]으로 기억에 박제돼 대충 되돌려지지 않음(soft lock = 미연시 PONR의 톡결 재해석).
  • 침묵·부재도 선택 공짜 — 오랜 부재를 캐릭터가 그리움·서운함·거리감으로 읽음(yEstranged). 메뉴형 VN이 절대 못 하는 몰입 수단.

9.진행 가시화 — 채팅은 순수, 대시보드는 옵트인 변형

9-1. 관계 육각 = 고정 육각 위 진도 오버레이 뷰어

roster 정적 stats[6](캐릭터 본성)은 그대로, 그 위에 NOTE:ME 파생 관계 육각(신뢰·설렘·편안·긴장·이해·역사) 오버레이. 프로필 시트(⚙·롱프레스)에만 — 채팅 스트림엔 절대 안 넣음.

왜: 6원소 배열이 이미 있으니 육각 위젯 재활용. RPG 스탯시트 쾌감을 옆칸에, 대화 방해 0.

9-2. 관계 단계 = 아바타 링/pip 뷰어

LV를 이름 옆 미세 링/점(등급 링 패턴)으로, bond 자라며 차오름. 채팅 안 숫자 대신 글랜서블 앰비언트 큐.

왜: 진행이 곁눈에 늘 보이되 대화 침범 0. 기존 링 UI 재활용.

9-3. 해금 목록 = 카드 뒷면(flip) 도감 뷰어

카드를 뒤집으면 해금 에피소드 체크리스트(잠긴 건 [LV3+] ??? 티저). 12카드 전원의 [LVn+] 4단을 그대로 목록화.

왜: “다음 목표”를 채팅 밖 지도로(공략집 감). 데이터는 이미 카드에.

9-4. 루트 지도 = 무음동 지도 위 관계선 뷰어

이미 있는 무음동 지도(좌표 확정·아이소메트릭 시안)에 내 관계선 오버레이(색·굵기=bond, 선 스타일=결). 루트 지도가 곧 동네 지도.

왜: 지도 에셋 재활용으로 문자 그대로의 루트 시각화 — 관계도+오픈월드 동시.

9-5. 채팅 내부 = 앰비언트로만 (지배 원칙) 공짜

채팅 안 진행은 오직 앰비언트 — bg 무드 전환·호칭 변화·문장 길이 성장·링 차오름. 팝업/토스트/숫자 금지. 하드 수치는 전부 옵트인 표면(프로필·앨범·지도). LV업 축하는 기존 소멸형 pill로 비모달.

왜: 이 원칙이 자유대화를 지키는 방어선. 대화는 순수, 게이지질은 대시보드로 격리 — “미연시인데 대화가 안 좁는다”의 실현점.

10.척추 — 이 설계의 유일한 기틀 변경 승인 필요

5축 전체의 척추 = NOTE:ME 서브라인 3종 신설 — [결:…](루트) · [칭호:…](엔딩) · [호칭:…](데레 마커). 이게 L0 출력 계약 골격 변경이라 유일하게 운영자 승인 + 5인 교차검증이 필요하다.

  • 이 한 스키마 확장만 승인되면 나머지 대부분은 뷰어/결정론(공짜·75% 위임 가능).
  • 보안 필수: 새 서브라인은 stripMarkers 화이트리스트 편입 + fresh 모델 출력(OUT)만 스캔(유저 hist/pending 스캔 금지 = 위조 주입 차단).
  • 비용: 서브라인은 같은 dispatch 안 토큰 몇 개 · 2번째 호출 0. 계기 시드도 기존 STATE_BLOCK 주입이지 새 호출 아님.
  • 자유대화·L0: 모델은 결·칭호를 강제 발화 안 함(전부 소프트 바이어스 + 채팅 밖 시각화). 해금 게이트는 이미 소프트.
