Ⅲ레이턴시 판정 근거(요구 ⑥⑧)

현 답장 체인 = 게이트웨이→GitHub Actions dispatch→claude -p→R2→폴. 웜 루프가 살아 있어도 후속 턴 십수 초, 콜드면 30초+. 어떤 튜닝으로도 "말하고 5초 내 음성 응답"은 이 체인에서 불가 → 운영자 기준(10초)에 따라 인앱 = 무전기 통신으로 명명·설계 전환(수신 무전 페이스: 텍스트 먼저→음성 수 초 뒤 자동재생). 진짜 전화 레이턴시(~1초대)는 상시 서버+스트리밍 스택이 필수라 Vapi 스캐폴드(10번 행)로 분리 — 두 축 모두 성립 = 요구 ⑨ "둘 다 불만족이면 FAIL" 회피.

Ⅳ리스크

  • 법적(최대): 보이스 클로닝은 본인 목소리 또는 권리 동의분만. 타인·연예인 음성 샘플 = 초상권·퍼블리시티권 침해. 파이프는 운영자 수동 dispatch 전용으로 잠가둠.
  • 과금 폭주: 무인증 공개 사이트에서 op ring/phone 남발 가능 → 일 상한(3통/2통·빈값도 보수 기본), TTS는 ptt·call 턴에서만 발동.
  • iOS 설치형 PWA: Web Speech 불가(실측 확인) → op stt 폴백. 단 AI 바인딩 미설정이면 무전은 타이핑 폴백으로 강등.
  • 규격 드리프트: Whisper turbo 입력 규격·브라우저 MediaRecorder mimeType 편차 → env 노브(YETA_STT_MODEL)와 관대한 파싱으로 완충.
  • 프라이버시: 음성(대사·샘플)은 전부 비공개 R2. 단 세션 op get 자체가 무인증 공개인 기존 이슈는 그대로(Zero Trust 잠금 옵션 §🗺).

추후 계획

아이데이션
  • 무드(<<MOOD>>)별 벨소리 톤 변주 · 새벽엔 DJ 윤이 "라디오 주파수"처럼 걸어옴(세계관 결합).

신의 한수
  • 음성을 공개 버킷이 아니라 비공개 세션 R2+게이트웨이 스트림으로 설계 — 대화 내용(음성) 공개 노출을 구조적으로 차단.
  • 텍스트 먼저·음성 나중 부착(ptt_voice) — TTS 시간이 체감 지연에 안 잡힘.
  • --safe-mode 기본화 — 토큰 절약(턴당 ~37k)이 곧 응답속도·쿼터 수명으로 직결(무전기 요구와 시너지).
  • WebAudio 재생(제스처 unlock 컨텍스트) — 모바일 자동재생 정책을 정면돌파.

  • Vapi assistant에 카드 전문을 넣었을 때 캐릭터 톤 유지 품질 · 국내 발신번호 표시/스팸필터 거동.
