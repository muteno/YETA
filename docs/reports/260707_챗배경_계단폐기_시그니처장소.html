한 줄 요약. 대화창을 열면 초록 네온 계단 포스터(yeta-bg.webp)가 base로 깔려 캐릭터 배경 로드 전/틈으로 비쳤다. 이걸 챗에서 완전 폐기(→ 검정 솔리드)하고, 배경을 「첫·오랜만 단독대화 = 처음 몇 발화는 캐릭터 고유 배경 → 이후 장소 배경, 평상시 연속대화 = 바로 장소」로 배선했다. 배경 로딩은 프리로드+캐시 즉시페인트로 챗 진입과 동기화.

왜 — 무엇이 문제였나

증상원인(실측)
대화창에 초록 계단이 겹쳐 보임("계단 오버레이")#yetadlg base 배경 = url(yeta-bg.webp) = 초록 네온 유리 계단 포스터. 캐릭터 배경(.ybg)은 92% 불투명이라 8% 비침 + 로드 전엔 100% 계단만 노출.
배경이 늦게 뜨는 애매한 화면yStage가 new Image() 비동기 프리로드 후 0.6s 페이드 → 텍스트보다 늦게 등장. 그 틈에 계단 base가 보였다.
첫 발화 직후 바로 장소로 전환기존 로직은 userN===0(내 발화 0)일 때만 시그니처, 첫 메시지부터 장소. "처음 몇 대화"가 없었다.

무엇을 했나 — 요구 4개 → 반영

요구반영상태
① 처음 몇 대화 = 캐릭터 고유 배경(프로필과 별개)새 ySigPhase() — 오프닝 국면 처음 Y_SIG_TURNS 유저턴은 YCUR.bg(캐릭터 시그니처 씬 · avatar와 별개 필드).반영
② 계단 오버레이 폐기(대화창 한정)#yetadlg base를 url(yeta-bg.webp)+스크림 → background:var(--bg) 검정 솔리드. 챗에서 계단·오버레이 비침 0. (포스터 파일은 존치 — 챗에서만 미사용)반영
③ 이후 = 장소 배경 · 첫/오랜만 아니면 바로 장소ySigPhase가 false면 ySceneForPlace 장소 씬. 평상시 연속대화(첫도 오랜만도 아님)는 오프닝 발화수가 이미 N 이상 → 즉시 장소. "단독"대화라 시그니처는 solo 전용(합석 2명이면 바로 장소 — 평의회 반영).반영
④ 배경 로딩 = 챗 진입과 동기2단 프리로드: ① 시그니처 bg 즉시 워밍(첫/오랜만 오프닝) ② 장소·무드 씬(36장) 유휴 지연 워밍 — 진행 중 대화(장소 국면) 재진입도 동기(평의회 BLOCK 반영). + 언락 즉시 워밍 착수 + yStage 캐시된 그림 즉시페인트(늦은 페이드 제거).반영

튜닝 노브 — 숫자 한마디로 조정

아래 두 값이 "몇 대화"·"오랜만"의 체감을 결정. viewer/index.html 상수라 즉시 조정 가능.

Y_SIG_TURNS = 3"처음 몇 대화" = 오프닝 시그니처 유지 유저 발화 수(미만이면 시그니처). 현재 = 인사 + 내 발화 1·2번째까지 시그니처 → 3번째 발화에서 장소로. "내 발화 3번까지 시그니처"를 원하면 4로.
