1. 무엇이 바뀌었나 (전/후)

항목전후
① 사진 저장 토스트"프로필 사진을 바꿨어" — 첫 첨부에도 '바꿨어'라 어색·저장 위치 맥락 없음"새 얼굴, 기억해뒀어 — 이 기기에만" (운영자 픽 = 무음동 캐릭터 톤 · 첨부/교체 겸용 · 기기 저장 맥락 동봉)
② 내 소개 행 오버플로소개가 길면 값(.si-v)이 전폭을 먹어 라벨이 "나…"·"캐…"로 압사(스샷 재현).si-v 최대 50% + .si-vv 말줄임 — 라벨 항상 온전·값은 초과분만 …
③ 카메라 배지배지가 원 안(overflow:hidden 클립) + 사진 첨부 시 has-img svg 규칙이 배지 글리프까지 숨겨 빈 라임 원(스샷 그 증상)배지 = 원 테두리 45° 걸침(right/bottom −3px) · has-img > svg로 좁혀 카메라 항상 유지 · 라벨 = 사진 없으면 "첨부"·있으면 "교체"
④ 능력치 다각형 상승분roster 고정 스탯만 — 친밀도 반영 없음yStatBoost(유효 친밀도 yBond 파생 · 온기/장난/텐션 축) → 상승분 = 뮤트 음영(--mut) 밑층 + 흰(--fg) 점선 둘레 윗층 = "다각형에 추가된" 표기 · 미대면/잠금 = 미표기 · 감쇠 시 자연 수축
⑤ PIN 변경전원 "서버 관리" 읽기전용관리자 = "관리자 계정은 불가"(env 전용 사유 표기) · 게스트 = 셀프 변경(새 PIN 입력 → op pinset → users.json 커밋 · 패턴 제약 1~9 무중복 4~8자리 검증)
⑥ 프로필 이미지 만들기없음(기기 사진 첨부만)프로필 없을 때 소개 행 아래 생성 게이지 행(.ycf-bar 정본 계승 · 120s 슬로우) → op meface → yeta-meface.yml → 소개 기반 로스터 톤 초상 → 공개 R2 yeta_face/me.png → sess.me.avatar CAS 주입 → 뷰어 폴 반영(기기 사진 > 생성분 우선순위)

2. 신규 파이프라인 구조

뷰어 게이지 행 → op meface{소개 존재 확인 · pending 3분 TTL · 일 상한 기본 2(YETA_MEFACE_MAX_PER_DAY) · 클라 텍스트 0} → yeta-meface.yml dispatch → yeta_meface.py{비공개 R2 세션에서 me만 읽음(대화 무접촉) · 스타일 = yeta_face.py BASE 임포트 = 로스터 톤 정합 · 공개 로그에 소개 원문 비출력} → 공개 R2 업로드 → 세션 ETag CAS 주입(러너 동형) → 뷰어 5s 폴 → 아바타 등장 + "프로필 이미지 나왔어" 토스트.
  • 비용 가드: 유료(OpenAI) 축 = ring/phone과 동일한 일 상한 계보. git 폴백 없음(개인화 이미지 레포 커밋 회피 — 공개 R2 필수).
  • PIN 셀프 변경: op pinset = 현재 PIN 실검증(auth 동일 해시) → users.json contents API 커밋(sha 낙관 락) · admin PIN 충돌/타 사용자 충돌 차단 = 권한 상승·계정 뒤섞임 원천 봉쇄 · 일 상한 기본 5.
  • op me 버그 픽스(동반): 종전 통짜 대입이 서버 관리 필드를 지움 → 스프레드 병합으로 me.avatar 보존.
