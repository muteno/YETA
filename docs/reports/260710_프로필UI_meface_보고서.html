2. PIN 셀프변경 = R2 오버라이드 (보안 재설계)

왜 재설계했나 — 초안은 게스트 PIN 변경을 users.json(공개 레포·main) 커밋으로 처리했다. 보안 평의회가 이를 치명 BLOCK — 무음동은 무인증 공개 게이트웨이라, 누구나 (1) 공개 레포에서 크랙한 게스트 PIN으로 (2) production main에 커밋을 유발(GH_TOKEN 남용)할 수 있었다. 운영자 확정 하에 R2 재설계로 전환.
  • 저장: 새 PIN은 users.json을 건드리지 않고 비공개 R2 auth/overrides.json = {원래해시: 새해시}에 CAS(etag if-match)로만 쓴다. GitHub 쓰기 축(GH_TOKEN) 완전 제거.
  • 인증: auth가 effectiveHash = overrides[pin_h] ‖ pin_h로 대조 → 새 PIN 로그인 + 원래 PIN 자동 무효화. 오버라이드 손상/부재 = 원래 해시로 fail-open(잠금 아님).
  • 무결성: 오버라이드 키는 항상 원래 pin_h(단조 = 사용자당 1엔트리, 무한증식 없음) · 유일 writer = pinset · admin PIN을 old/next로 = 차단 · 타 사용자 effectiveHash 충돌 차단.
  • 브루트포스: 시도(성공·실패 무관) 즉시 상한 소비(종전 성공만 카운트하던 구멍 봉합) · 일 상한 기본 5.
  • op me 동반 버그픽스: 종전 통짜 대입이 서버 관리 필드를 지움 → 스프레드 병합으로 me.avatar 보존.
잔여 = 게스트 PIN이 공개 레포에서 크랙 가능(=보안벽 아님)한 것은 선재·운영자 수용. R2 재설계는 그 위에서 main 커밋 유발 프리미티브를 제거한 것이 핵심.

3. meface 파이프라인 + 동시성

뷰어 게이지 → op meface{소개 존재·avatar 미보유 확인 · pending 선점을 casPut 원자화(동시 버스트 단일 통과) · 일 상한 기본 2 · 클라 텍스트 0} → yeta-meface.yml → yeta_meface.py{비공개 R2에서 me만 읽음(대화 무접촉) · 스타일 = yeta_face.py BASE 임포트 = 로스터 톤 · 공개 로그에 소개 원문 비출력 · git 폴백 없음} → 공개 R2 → 세션 ETag CAS 주입 → 뷰어 폴(me만 병합 · 185s 타임아웃 = 서버 pending 180s 정렬) → 아바타 등장.
