2계승 원칙 — 신규 창작 0

값 SSOT(:root) 무변경. 글래스 표면은 전부 기존 토큰 계승으로 구현 — --glass·--glass-2·--glass-line·--blur-m·--blur-l·--r-modal·--shadow-card·--bg-scrim. 새 색·px·blur·토큰 0개. 확립 패턴 수신 콜 화면(.ycall-bg 배경 + .ycall-line 글래스 자막)을 챗 전면으로 확장.

  • --bubble-ai 토큰은 @supports not (backdrop-filter) 불투명 폴백으로 배선 유지 — 죽은 토큰 0, 구형 브라우저 폴백 겸용.
  • 브랜드 계약 준수 — 강조는 라임 1색만(내 버블), raw 강조색 0. 나머지 흰/글래스.
  • 페르소나 무대 .ybg = 라임 글래스 base 위 은은한 무드 베일(기틀 <<MOOD>>·yeta_bg.py 보존).

3검증 — 자가검증 통과

  • check_refs rc=0 — 토큰 락·디자인 게이트·거울 정합·구문 전부 통과. 신규 raw hex/blur/accent = 0.
  • Playwright 실측 2샷 — 로컬 서빙 + 샘플 세션 주입, 390×844@2x 렌더(위 스크린샷). 화면·추측 아닌 실제 렌더 확인.
  • 정본 동기화 — CII 인덱스 행 42(#yetadlg)·40(버블) 갱신, 작업이력 append.

4튜너블 — 운영자 한마디로 조정
