친밀도를 음수(역행)로 — 대칭 확장 설계 정본

"하한 0" 은 하한이 아니라 발화 수 카운트의 부산물이다. 진짜 필요한 건 관계-손상이라는 새 축이다.

접수 2026-07-12 17:17 KST 평의회 아키텍처·신호원·수학·tune·UX·레드팀 (6/6 수렴) 운영자 결정 하이브리드 트리거 · 별도 채널 · 전원 on 상태 설계 확정 · 구현 전(5인 기틀검증 대기)

결론. 이건 yBond 한 줄 수정이 아니다. yBond = 내 발화 수(n) × 감쇠 는 n이 카운트라 수학적으로 음수가 안 나온다 — 0은 하한이 아니라 "발화가 없다"는 뜻이다. 역행을 만들려면 페르소나가 emit하는 부호있는 감점(grudge) 이라는 새 영속 축을 신설해야 하고, 그 순간 세션 스키마·러너·뷰어를 건드리는 기틀 변경이 된다.

네가 고른 3개 분기 위에서 6인이 수렴한 설계가 아래다. 감쇠는 같은 수식을 부호있는 값에 그대로 걸어 "동일 개념"을 보존한다.

1 진단 — 왜 지금 0이 하한인가

현재 viewer/index.html:2947 의 유효 친밀도:

function yBond(c){
  const n = (th.turns||[]).filter(t=>t.role==='user').length;  // 내 발화 수 (≥0)
  const ease = smoothstep(1일→7일 방치);                     // 0 → 1
  return n * (1 - .5*ease);                              // 승수 ∈ [0.5, 1]
}

n ≥ 0 이고 감쇠 승수는 [0.5, 1] 이라 곱은 절대 음수가 못 된다. 방치해도 0.5·n(절반)까지만 식고 0으로도 안 간다. 즉 "악화"라는 개념이 코드에 아예 없다 — 지금은 나쁜 대화조차 발화 수를 올린다. 소원(yEstranged)도 원인은 오직 방치지 갈등이 아니다.

2 네가 고른 3개 분기 (설계 확정)

① 트리거 = 하이브리드

페르소나가 진짜 모욕·반복 냉대를 감지하면 <<BOND:−n>> 마커 emit(의미 판단) + 합석 거절·강퇴·장기 무시 같은 기계 규칙이 결정적 바닥. "말로 준 상처"도 잡고 일관성도 확보.

② 역사 버퍼 = 별도 채널

감점을 발화 수와 네팅하지 않는다. 활성 감점이 있으면 그게 관계를 지배 — 배신은 대화 역사와 무관하게 깊게 벤다. 단 역사(n)는 밑에 보존돼, 감점이 풀리면 원래 관계로 복귀.

③ 적용 = 전원 기본 on

10인 전원 역행 상시 적용. 가장 리얼하지만 "앱이 불쾌·차단될" 리스크가 실재 → §5의 온기 가드가 필수 짝(음수 = 냉담·거리 축이지 학대 축 아님).

기각된 것

영구 "절연" 계층 — 말벗 앱에 영구 단절은 제품 자살(UX 위원). 회복로는 항상 존재한다. rel_lv 재사용 — 부호없는 한자리·§🔓 게이트축과 충돌(아키텍처 위원).

3 대칭 수학 — 같은 감쇠, 부호만 보존

"기존 알고리즘과 동일 개념"의 핵심: 감쇠는 원래 raw에 걸리는 승수다. 부호있는 값에 그대로 곱하면 양·음이 자동 대칭이 된다. 시간은 원한을 0으로 지우지 않고 절반까지만 식힌다(양수 반감과 동일). 완전 회복은 시간이 아니라 재회·사과 이벤트가 한다(소원 회복이 이미 이벤트 기반인 것과 정합).

양수 = 발화 수 × 감쇠 (기존 그대로) 음수 = 감점 × 같은 감쇠 (신규·부호 보존)

제안 수식 (별도 채널 확정판)

function yBond(c){
  const th = ((YSESS||{}).threads||{})[c&&c.id]; if(!th) return 0;
  const n = (th.turns||[]).filter(t=>t.role==='user').length;
  const g = Math.max(-12, (+th.grudge||0));      // ★신규 감점 채널 ≤0 · 바닥 −12 · undefined→0 (NaN 가드)
  if(!n && !g) return 0;
  const d = Math.max(0,(Date.now()-(th.updated||0))/86400e3);
  const t = Math.min(1,Math.max(0,(d-1)/6)), ease = t*t*(3-2*t);
  const decay = 1 - .5*ease;                       // 양·음 공용 승수 [0.5,1]
  return (g < 0 ? g : n) * decay;              // ★별도 채널: 감점 살아있으면 감점이 지배(역사 무관)
}                                                //  g 풀리면(→0) 발화 채널 복귀 = 역사 보존

왜 별도 채널이 우아한가: 관계 역사 n 은 파괴되지 않고 밑에 그대로 살아있다. 감점 g 가 위에 음수로 덮여 관계를 가리고, 재회·사과로 g→0 이 되면 n·decay 양수 관계가 그대로 되살아난다. "깊게 베이되 영구 파괴는 아니다"가 수식 한 줄에 들어간다.

육각 스탯도 대칭 — 역행하면 수축한다 운영자 추가 260712

지금 yStatBoost(index.html:2265)는 친밀도가 오르면 온기·장난·텐션 축을 base 위로만 키운다 — 육각이 커지고 상승분이 음영(--mut)으로 삐져나온다(점진). 네 지적대로 이건 단방향이다. 대칭으로 반대 방향을 넣는다: bond가 음수로 가면 같은 축이 base 밑으로 수축해 육각이 눈에 띄게 쪼그라든다. 최근 역행일수록(감쇠가 최근성을 이미 반영) 더 줄어든다.

라임 폴리곤 = 유효 스탯 흰 점선 = 성향 base(고정) 수축 = base 밑으로 탈채도 음영(잃은 온기·장난) 
function yStatBoost(c){
  if(!c || !Array.isArray(c.stats) || c.stats.length!==6 || yUnmet(c) || yLockLabel(c)) return null;
  const b = yBond(c), W=[1,.7,0,0,.55,0];               // 온기·장난·텐션 축만
  if(b >= 2.5){                                    // 상승(기존) — 정서축 성장
    const k=Math.min(1,b/12);
    const out=c.stats.map((v,i)=>Math.min(5, v + 1.3*k*W[i]));
    return out.some((v,i)=>v-c.stats[i]>.05) ? out : null;
  }
  if(b < 0){                                       // ★역행(신규) — 같은 축이 base 밑으로 수축
    const k=Math.min(1,-b/12);                        // |b|/12 · 감쇠가 최근성 반영
    const out=c.stats.map((v,i)=>Math.max(0, v - 1.3*k*W[i]));    // 하한 0(스탯 바닥)
    return out.some((v,i)=>c.stats[i]-v>.05) ? out : null;
  }
  return null;                                      // 0~2.5 = 중립(델타 없음)
}

렌더 주의(§🎨): yHexSvg(2253)는 지금 상승분만 base 바깥으로 그린다("삐져나온 델타만 보임"). 수축은 유효 폴리곤이 base 안쪽이라, 잃은 영역(base와 유효 사이)을 탈채도 음영으로 채우는 렌더 분기가 필요하다 — 색은 반목과 같은 --mut 계승(새 색 창작 금지). 이 표기는 플레이그라운드로 전·후 실물 확정 후 배선.

4 관계 계층 — 소원 밑에 "반목" 딱 1개

결정적 함정: 음수 bond가 그냥 들어가면 yEstranged(bond<1.5) 를 참으로 뒤집어 소원=탭 차단으로 삼켜버린다. 그건 "악화됐지만 대화는 되는 반목"의 정반대다. 반목은 차단 아님(냉담하게라도 대화는 됨)으로 별 술어 분리가 필수.

5 역행 페르소나 — tune 축 매핑

선례가 이미 있다: yReadMs 가 tune[7](삐짐) 을 계수 140의 부정축으로 쓴다. 같은 형식으로 역행 속도·깊이·회복을 성향에서 파생한다. 경계심(11)은 제외 — 그건 접근 게이트(yFarNeed) 전용이라, 재사용하면 "차갑지만 의리형"(백)이 이중 페널티로 오분류된다.

// 나쁜 턴 감점 배율 — 삐짐(7)·낙차(8)↑ 쉽게·깊게 / 인내심(6)·온기(5)↑ 관대
function yFallRate(c){ const g=yTuneCur(c);
  return Math.max(.3, Math.min(2.5, 1 + g[7]*.12 + g[8]*.06 - g[6]*.09 - g[5]*.05)); }
// 회복 배율 — 온기·인내심↑ 용서 빠름 / 삐짐·경계심↑ 뒤끝
function yThawMul(c){ const g=yTuneCur(c);
  return Math.max(.4, Math.min(2, 1 + (g[5]+g[6])*.06 - (g[7]+g[11])*.05)); }
역할캐릭터tune 근거거동
역행형세라삐짐7=6 · 낙차8=10 · 인내6=3가장 쉽게·일찍 식음 (yFallRate≈1.70)
역행형루시삐짐7=7 · 낙차8=10 · 인내6=3낙차 최대 + 뒤끝(yThawMul≈0.78)
회복 비대칭하서연삐짐7=7 이나 인내6=7·온기5=9"쉽게 삐지고 금방 푸는" 표본
관대형홍석천·백삐짐7=0 · 인내6=8웬만해선 안 틀어짐 (클램프 .3 바닥)
