PIN 게이트 — .ymain.pin-mode

접속 CTA·잠긴 탭이 데려오는 메인 화면의 2번째 상태(별도 라우트 아님). 워드마크가 40px로 줄며 상단 고정되고, 4슬롯 + 3×4 키패드가 올라온다. 검증 = SHA-256(pin+':yeta') 로컬 대조, 5회 오류 = 10분 잠금, 성공 = 네비 전체 활성(.unlocked). 정본 = viewer/index.html CSS L217~252 · JS L581~625.

📍 값 SSOT = viewer/index.html :root → base.css 거울 계승 · PIN 강조 = --pin-ac(= var(--accent-2))·오류 = --m-danger · 브랜드 = 네온레몬 --brand(적용 검토 §4).

2 상태머신

입력 중(0~4자리)→ 4자리 자동판정 성공 = 해제/ 오류 (n/5)→ 5회 10분 잠금
상태화면 반응영속(localStorage)
입력 중슬롯 차오름(.on) · 키 탭 플래시 · 4자리째 즉시 해시 대조(확인 버튼 없음)—
오류슬롯 4칸 .bad(링·별 --m-danger) + lkShake .45s + "PIN 번호가 틀립니다 (n/5)" → 1.4s 후 자동 비움yeta_pin_fails 누적
잠금"PIN 오류 5회 — N분 후 다시" · 키 입력 무시(메시지만 갱신)yeta_pin_until = now+10분
성공초록 대신 강조1 문구 "잠금 해제 — 메뉴 활성"(.lk-ok) · .unlocked 부여 → 네비 잠금 해제yeta_pin_ok=1 · fails/until 삭제
재방문yeta_pin_ok=1이면 로드 즉시 해제(키패드 생략 — 소프트 게이트)읽기

보안 결: 해시 = crypto.subtle.digest('SHA-256', pin+':yeta') — 평문 PIN 비저장. 이건 UI 소프트 게이트(연출·오조작 방지)일 뿐 서버 게이트가 아님 — 서버측은 Cloudflare Access 부착이 정본(CLAUDE.md §🔑). reduced-motion = shake 즉시 종료.
