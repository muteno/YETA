2 상태 필름 (기틀 실값 · 문구 원문)

무무디 · 입력 중

① 대기 = 타이핑 버블

.ytyp — 점 3개 바운스(1.15s 순차). 실제 버블로 위장(카톡 결) · 오래 걸려도 "입력 중" 확신. 등장만 촤르륵, 폴 재렌더는 무애니.

무무디
천천히 마셔. 아무도 안 뺏어.27s

② 답장 착지

타이핑 버블 자리 → 캐릭터 버블 촤르륵 + 캡션(소요초 tabular). 폴 정지 · state=idle.

오늘 좀 길었어
네트워크 실패 — 다시 보내줘

③ 전송 실패(즉시)

fetch 실패 = 낙관 노드 촤르륵 회수(ybOut 접힘) + 입력창에 원문 복원(재타이핑 0) + 실패 버블.

답장을 못 만들었어 — 다시 보내면 재시도야. (quota)

④ 생성 실패 state=error

러너 실패(에러 80자 동봉). [다시 보내기] = retry op — 기존 pending 턴 재발사(새 턴 추가 X). 실패 시 miniToast "재시도 실패 — 잠시 후 다시".

답장이 너무 늦네 — 러너가 밀렸거나 실패한 듯. 다시 보내면 재시도야.

⑤ 좀비(5분 초과)

awaiting_since +5분(zlim 300000 · effort low 고정) 넘으면 대기 중단 표시 + 폴 정지 — 가짜 실패로 인한 중복 쿼터 방지 임계.

⚠️ 저장소(R2) 미설정 — 셋업 후 대화가 켜져.

⑥ R2 미설정

setup 응답 = 안내만 + 입력행 숨김. 리스트에서 진입 시엔 소개 배너 + "소개만 볼 수 있어" — 구경은 항상 가능(chars는 R2 불필요).
